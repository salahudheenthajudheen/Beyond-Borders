# Contribution Review Checklist

> Use this checklist before opening a pull request. Reviewers will evaluate contributions against these criteria.

---

## Before You Submit

Run through this checklist and confirm each item is complete. Incomplete submissions will be returned for revision before review.

---

## Problem Statement Checklist

### Research Quality
- [ ] I have spoken to at least 3 people who face this problem directly (not assumed it exists)
- [ ] I have documented the scale of the problem (number of people affected, economic/social cost)
- [ ] I have verified that no identical problem statement already exists in this repository
- [ ] I have cited at least 2 credible external sources

### Solution Quality
- [ ] My proposed solution specifically leverages the named company's technology (not a generic solution)
- [ ] I have explained specifically why Edge AI is necessary (not just beneficial)
- [ ] My MVP is realistic — achievable in 8–16 weeks with a small team
- [ ] I have identified the minimum data requirements to build the solution

### Evaluation Rigor
- [ ] My Desirability, Feasibility, and Viability scores are backed by evidence in the document
- [ ] I have been honest about risks and challenges — not just listed token risks
- [ ] My market sizing (TAM/SAM) is based on cited data, not guesses
- [ ] My startup potential score reflects the weakest dimension, not only the strongest

### Format and Quality
- [ ] My file uses the correct template structure
- [ ] My file name is lowercase kebab-case (e.g., `precision-irrigation-edge-ai.md`)
- [ ] My file is in the correct folder path
- [ ] No spelling or grammar errors (run a spellcheck)
- [ ] Language is clear and globally understandable — no regional jargon without explanation

---

## Company Profile Checklist

- [ ] Company overview is factually accurate (verified from official sources)
- [ ] Technology description is correct and specific — not marketing language
- [ ] Innovation map is complete for at least one vertical
- [ ] At least 3 problem statements exist before submitting a company profile
- [ ] All links in the document are functional

---

## Project Checklist

- [ ] Project addresses a specific problem statement in this repository (linked)
- [ ] Technical approach is specific — not "build an AI model"
- [ ] Milestones are time-bound and realistic
- [ ] "How to Contribute" section gives enough detail for a new contributor to start
- [ ] Skills needed section is honest about what is required

---

## General Standards

- [ ] Content is original — not copied from other sources
- [ ] No placeholder text left in the template (e.g., "_Description here_")
- [ ] No AI-generated filler that adds length without adding insight
- [ ] Every claim that can be verified is cited

---

## Reviewer Guidance

Reviewers evaluate submissions on:

1. **Depth** — Does this show real understanding of the domain?
2. **Honesty** — Are risks and limitations acknowledged?
3. **Specificity** — Is the solution tied to the specific technology, or generic?
4. **Actionability** — Can someone pick this up and build from it?
5. **Fit** — Does this belong in Beyond Borders, or is it better placed elsewhere?

Reviewers should provide constructive, specific feedback — not just "reject." The goal is to help contributors improve submissions to the required standard.
