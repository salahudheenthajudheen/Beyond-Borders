# Contributing to Beyond Borders

Thank you for contributing to Beyond Borders. This repository grows through the efforts of students, founders, researchers, and ecosystem builders around the world. Every contribution you make helps someone, somewhere discover an opportunity they wouldn't have found otherwise.

---

## What You Can Contribute

- **Problem Statements** — A structured, well-researched problem in a specific vertical
- **Use Cases** — Detailed application of a frontier technology to a real-world domain
- **Project Ideas** — Open-source projects others can build and contribute to
- **Startup Opportunities** — Validated startup concepts with market analysis
- **Research Directions** — Open questions and research gaps worth investigating
- **Company Profiles** — A new company entry following the company template
- **Framework Improvements** — Better ways to evaluate ideas, score startups, or review contributions

---

## Contribution Standards

Beyond Borders is a serious innovation knowledge base. Contributions must meet these standards:

### Quality over Quantity
One deeply researched problem statement is worth more than ten shallow ones. Do the work. Understand the domain. Talk to people who face the problem. Ground your analysis in reality.

### No Random Demo Ideas
Every problem statement must have genuine desirability, feasibility, and viability. If you wouldn't stake 2 years of your life on this, don't submit it.

### Use the Templates
All contributions must follow the relevant template:
- [Use Case Template](./usecase-template.md)
- [Company Template](./company-template.md)
- [Project Template](./project-template.md)
- [Startup Template](./startup-template.md)

### File Naming
Use lowercase kebab-case for all file names. Examples:
- `precision-irrigation-intelligence.md` ✅
- `Precision_Irrigation.md` ❌

## How to Contribute

### Step 1: Fork the Repository
Fork this repository to your GitHub account.

### Step 2: Create a Branch
Use a descriptive branch name:
```
git checkout -b add/netrasemi-healthcare-ai-ps4
```

### Step 3: Create Your Contribution
Follow the appropriate template. Put your file in the right folder. Check the existing files to understand the expected depth and tone.

### Step 4: Self-Review
Before opening a PR, run through the [Contribution Review Checklist](./frameworks/contribution-review-checklist.md).

### Step 5: Open a Pull Request
Open a PR with:
- A clear title describing what you're adding
- A short description of your research process
- Any sources or references you used

## Adding a New Company

To add a new company to Beyond Borders:

1. Create a folder: `companies/<company-name-lowercase>/`
2. Copy and fill in the [Company Template](./company-template.md) as `README.md`
3. Add `company-overview.md` and `technology-overview.md`
4. Create subfolders: `usecases/`, `projects/`, `startups/`, `research/`
5. Add at least 3 problem statements under one vertical before opening a PR
6. Open a PR titled: `[Company] Add <Company Name>`
